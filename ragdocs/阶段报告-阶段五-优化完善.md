### 1.2 功能完善（级联删除）

**问题**：删除笔记本时只删除了笔记本本身，关联的来源、报告、测验、答题记录和向量存储文件都成了"孤儿数据"。

**方案**：在 `NotebookServiceImpl` 中新增 `deleteNotebookCascade()` 方法，使用 `@Transactional` 事务注解确保数据一致性：

```
删除顺序：答题记录 → 测验 → 报告 → 来源 → 向量存储文件 → 笔记本
```

## 3. 思考过程

### 3.1 为什么 URL 不能硬编码？

开发环境是 `localhost:8123`，但部署到服务器后端口和域名会变。使用 `window.location.origin` 可以自动适配任何环境。

### 3.2 为什么 `calculateScore` 不能放在普通 `<script>` 块中？

Vue 3 的 `<script setup>` 和普通 `<script>` 有不同的作用域。`ref()` 创建的响应式变量只在 `<script setup>` 中可用，在普通 `<script>` 块中访问 `.value` 会因为变量不在同一作用域而出错。

### 3.3 为什么级联删除要用事务？

如果删除过程中某一步失败（比如删除来源时数据库异常），没有事务的话已删除的数据无法恢复，导致数据不一致。`@Transactional(rollbackFor = Exception.class)` 确保要么全部成功，要么全部回滚。

### 3.4 为什么 `QuizRecord` 的 ID 策略很重要？

数据库表定义了 `auto_increment`，但 Java 实体用了 `snowFlakeId`。如果两边不一致，插入时数据库不会自动生成 ID（因为 snowFlake 会在 Java 端生成一个大数），可能导致 ID 冲突或插入失败。
