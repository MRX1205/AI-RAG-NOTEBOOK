| 精选详情 | GET | `/public/notebooks/{id}` | 返回单本详情 + 来源列表，仅精选可访问 |

```java
    // GET /featured - 无需鉴权
    // GET /{id} - 无需鉴权，仅 isFeatured=1 可访问
}
```

#### 克隆接口（需登录）

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 克隆笔记本 | POST | `/notebook/{id}/clone` | 为当前用户创建副本，返回新笔记本 ID |

克隆逻辑（`NotebookServiceImpl.cloneNotebook`）：
1. 检查源笔记本存在且 `isFeatured=1`
2. 创建新笔记本（标题加「副本」前缀，`isFeatured=0`）
3. 复制所有 `Source` 记录，更新 `notebookId` 和 `userId`，时间字段使用当前时间
4. 更新新笔记本的 `sourceCount`
5. 复制向量存储文件（`VectorStoreConfig.copyStore(sourceId, targetId)`），使 RAG 立即可用

#### 向量存储复制（VectorStoreConfig.copyStore）

```java
public void copyStore(Long sourceNotebookId, Long targetNotebookId) {
    persistStore(sourceNotebookId);  // 先持久化源笔记本
    Files.copy(sourcePath, targetPath, REPLACE_EXISTING);
    // 副本即可立即使用 RAG 检索，无需重新 Embedding
}
```

## Bug 修复记录

**Bug**：克隆接口返回 50000（系统错误）

**原因**：`cloneNotebook` 中设置 `newSource.setCreateTime(null)`，
`sourceMapper.insert()` 生成的 SQL 显式传入 NULL，违反 `createTime NOT NULL` 约束。

**修复**：改为 `newSource.setCreateTime(LocalDateTime.now())`
