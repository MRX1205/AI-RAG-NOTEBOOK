## Bug 1 — 修复硬编码 URL

### 问题是什么？

在 `chatController.ts` 和 `StudioPanel.vue` 中，请求 URL 写死了 `http://localhost:8123/api`：

```typescript
// ❌ 问题代码：只能在本地开发环境工作
let url = `http://localhost:8123/api/chat/stream?notebookId=${notebookId}...`
```

如果项目部署到服务器（如 `https://example.com`），这些请求就会失败。

### 怎么修？

使用 `window.location.origin` 动态获取当前域名和端口：

```typescript
// ✅ 修复后：自动适配任何环境
const baseUrl = `${window.location.origin}/api`
let url = `${baseUrl}/chat/stream?notebookId=${notebookId}...`
```

### 修改文件

📄 **lyhm-ai-rag-frontend/src/api/chatController.ts**

```typescript
export function getChatStreamUrl(notebookId: number, message: string, sourceIds?: number[]) {
    // 动态获取当前域名，不再硬编码 localhost
    const baseUrl = `${window.location.origin}/api`
    let url = `${baseUrl}/chat/stream?notebookId=${notebookId}&message=${encodeURIComponent(message)}`
    if (sourceIds && sourceIds.length > 0) {
        url += `&sourceIds=${sourceIds.join(',')}`
    }
    return url
}
```

📄 **lyhm-ai-rag-frontend/src/components/notebook/StudioPanel.vue**

```typescript
// 报告生成的 fetch 请求
const response = await fetch(`${window.location.origin}/api/report/generate`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body),
    credentials: 'include',
})
```

### `window.location.origin` 是什么？

浏览器内置属性，返回当前页面的协议 + 主机名 + 端口：
- 本地开发：`http://localhost:5173`
- 生产环境：`https://example.com`
- 带端口：`http://192.168.1.100:8080`

## Bug 2 — 修复 calculateScore 响应式引用

### 问题是什么？

`StudioPanel.vue` 有两个 `<script>` 块：

```html
<!-- 块1：Composition API（响应式变量定义在这里） -->
<script setup lang="ts">
const currentQuiz = ref<API.QuizVO | null>(null)   // 响应式
const quizAnswers = ref<Record<number, string>>({}) // 响应式
</script>

<!-- 块2：普通 Options API（calculateScore 在这里） -->
<script lang="ts">
function calculateScore(): number {
    // ❌ 这里访问不到 setup 块中的 currentQuiz 和 quizAnswers
    if (!currentQuiz.value?.questions) return 0
}
</script>
```

在 Vue 3 中，`<script setup>` 内定义的变量对普通 `<script>` 块是不可见的。

### 怎么修？

把 `calculateScore` 移到 `<script setup>` 块中：

```typescript
// 在 <script setup> 块中
/** 计算得分（百分制） */
const calculateScore = (): number => {
  if (!currentQuiz.value?.questions) return 0
  try {
    const questions: API.QuizQuestion[] = JSON.parse(currentQuiz.value.questions)
    if (questions.length === 0) return 0
    let correct = 0
    for (const q of questions) {
      if (q.questionId && quizAnswers.value[q.questionId] === q.correctAnswer) {
        correct++
      }
    }
    return Math.round((correct / questions.length) * 100)
  } catch {
    return 0
  }
}
```

### Vue 3 双 script 块的规则

| 场景 | 使用哪个块 |
|------|-----------|
| 定义响应式数据、方法 | `<script setup>` |
| 导出非 setup 的函数（如 `renderMarkdown`） | 普通 `<script>` |
| 需要访问 `ref` / `reactive` 变量的函数 | **必须**在 `<script setup>` 中 |

## Bug 3 — 修复 QuizRecord ID 策略不匹配

### 问题是什么？

数据库 `quiz_record` 表使用自增 ID：

```sql
id bigint auto_increment comment 'id' primary key
```

但 Java 实体类使用了雪花算法：

```java
// ❌ 不匹配：数据库是 auto_increment，Java 是 snowFlakeId
@Id(keyType = KeyType.Generator, value = KeyGenerators.snowFlakeId)
private Long id;
```

这会导致插入时 Java 端生成 snowFlake ID（如 `1896543210987654321`），但数据库也想自增生成 ID，两边冲突。

### 怎么修？

📄 **src/main/java/com/lyhm/airag/model/entity/QuizRecord.java**

```java
// ✅ 改为自增，与数据库表定义一致
@Id(keyType = KeyType.Auto)
private Long id;
```

同时移除不再需要的 import：

```diff
-import com.mybatisflex.core.keygen.KeyGenerators;
```

### ID 生成策略对比

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| `KeyType.Auto` | 数据库自增 | 单机环境，简单可靠 |
| `KeyType.Generator` + `snowFlakeId` | 雪花算法生成分布式唯一 ID | 分布式系统，需全局唯一 |
| `KeyType.Generator` + `flexId` | MyBatis-Flex 内置 ID 生成 | 类似雪花算法的替代方案 |

**一致性原则**：数据库表定义了什么策略，Java 实体就要用什么策略。

## 功能 — 实现级联删除

### 为什么需要级联删除？

之前删除笔记本时，只删除了 `notebook` 表中的记录。但数据库中还有大量关联数据：

```
notebook (id=123)
  ├── source (notebookId=123) → 3 条来源记录
  ├── report (notebookId=123) → 2 份报告
  ├── quiz (notebookId=123)   → 1 份测验
  │     └── quiz_record (quizId=456) → 5 条答题记录
  └── vector-store/notebook_123.json → 向量存储文件
```

如果不清理，这些数据会变成"孤儿数据"，浪费存储空间，甚至可能导致查询异常。

### Step 1 — NotebookService 接口新增方法

📄 **src/main/java/com/lyhm/airag/service/NotebookService.java**

```java

 * 级联删除笔记本及其关联数据

 */

```

### Step 2 — NotebookServiceImpl 实现级联删除

📄 **src/main/java/com/lyhm/airag/service/impl/NotebookServiceImpl.java**

```java
@Slf4j  // 新增日志注解
@Service
public class NotebookServiceImpl extends ServiceImpl<NotebookMapper, Notebook>
        implements NotebookService {

    // 注入关联表的 Mapper
    @Resource
    private SourceMapper sourceMapper;
    @Resource
    private ReportMapper reportMapper;
    @Resource
    private QuizMapper quizMapper;
    @Resource
    private QuizRecordMapper quizRecordMapper;
    @Resource
    private VectorStoreConfig vectorStoreConfig;

    // ... 原有的 getNotebookVO 和 getNotebookVOList 方法保持不变 ...

    /**
     * 级联删除笔记本及所有关联数据
     *
     * @Transactional 确保要么全部删除，要么全部回滚
     */
    @Override
    @Transactional(rollbackFor = Exception.class)
    public void deleteNotebookCascade(Long notebookId) {
        log.info("开始级联删除笔记本: {}", notebookId);

        // 1. 先删除测验的答题记录（依赖 quiz 表）
        List<Quiz> quizList = quizMapper.selectListByQuery(
                QueryWrapper.create().eq("notebookId", notebookId));
        for (Quiz quiz : quizList) {
            quizRecordMapper.deleteByQuery(
                    QueryWrapper.create().eq("quizId", quiz.getId()));
        }

        // 2. 删除测验
        quizMapper.deleteByQuery(
                QueryWrapper.create().eq("notebookId", notebookId));

        // 3. 删除报告
        reportMapper.deleteByQuery(
                QueryWrapper.create().eq("notebookId", notebookId));

        // 4. 删除来源
        sourceMapper.deleteByQuery(
                QueryWrapper.create().eq("notebookId", notebookId));

        // 5. 删除向量存储文件
        vectorStoreConfig.removeStore(notebookId);

        // 6. 最后删除笔记本本身
        this.removeById(notebookId);

        log.info("笔记本 {} 级联删除完成", notebookId);
    }
}
```

**删除顺序很重要**：
```
quiz_record → quiz → report → source → vector_store → notebook
（先删子表，再删父表，避免外键冲突）
```

### Step 3 — 更新 Controller

📄 **src/main/java/com/lyhm/airag/controller/NotebookController.java**

```java
@PostMapping("/delete")
public BaseResponse<Boolean> deleteNotebook(@RequestBody DeleteRequest deleteRequest,
        HttpServletRequest request) {
    // 参数校验 + 权限校验（不变）...

    // ✅ 改为级联删除（替换原来的 removeById）
    notebookService.deleteNotebookCascade(deleteRequest.getId());
    return ResultUtils.success(true);
}
```

### 关键注解：@Transactional

```java
@Transactional(rollbackFor = Exception.class)
```

- **作用**：把整个方法包裹在一个数据库事务中
- **rollbackFor = Exception.class**：任何异常都触发回滚（默认只对 RuntimeException 回滚）
- **效果**：如果第 3 步删报告失败，第 1-2 步已删除的答题记录和测验也会恢复

## 本阶段知识点总结

| 知识点 | 说明 |
|--------|------|
| **window.location.origin** | 浏览器 API，动态获取当前域名+端口，避免硬编码 |
| **Vue 3 双 script 块** | `<script setup>` 和 `<script>` 作用域隔离，响应式变量只在 setup 中可用 |
| **数据库 ID 策略一致性** | Java 实体的 `@Id` 策略必须与数据库表定义一致 |
| **级联删除** | 删除父记录时清理所有子记录，避免孤儿数据 |
| **@Transactional** | Spring 事务管理，保证多步操作的原子性（全成功或全回滚） |
| **删除顺序** | 先删子表再删父表，按依赖关系从下往上逐层清理 |
