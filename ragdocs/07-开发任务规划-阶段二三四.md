# 阶段二~四 开发任务规划文档

> 本文档记录了开发阶段二（知识库核心）、阶段三（AI 对话）、阶段四（报告 + 测验）的详细任务规划和思考过程。

## 一、总体思路

### 1.1 项目目标
实现一个类似 Google NotebookLM 的 AI 知识库系统，核心功能包括：
- **来源管理**：用户上传文档/粘贴文本 → 解析 → 分块 → 向量化 → 存入向量数据库
- **RAG 对话**：用户提问 → 检索相关文档片段 → 作为上下文发给 LLM → 流式返回答案
- **报告生成**：基于文档内容生成多种类型报告（简报/学习指南/FAQ/时间线/自定义）
- **测验生成**：基于文档内容生成选择题 → 用户答题 → 自动评分

### 1.2 技术选型思考

| 决策点 | 选型 | 原因 |
|-------|------|------|
| AI 框架 | LangChain4j | Java 生态最成熟的 LLM 应用框架，提供文档加载、分块、嵌入、向量存储等完整链路 |
| 嵌入模型 | DashScope text-embedding-v3 | 阿里通义系列，中文效果好，API 稳定 |
| 对话模型 | DeepSeek | 性价比高，支持流式输出 |
| 向量存储 | InMemoryEmbeddingStore | 开发阶段够用，配合 JSON 持久化，避免部署外部服务（如 Milvus） |
| 文档解析 | Apache Tika | 支持 PDF/DOCX/TXT/MD 等多种格式 |
| 流式输出 | SSE (SseEmitter) | 适合 AI 逐字输出的场景，前端用 EventSource 接收 |

### 1.3 架构分层
```
前端 (Vue3 + Ant Design Vue)
    ↓ HTTP/SSE
后端 Controller 层（参数校验、权限验证）
    ↓
Service 层（业务逻辑）
    ↓
AI 层（RagService + LangChain4j）
    ↓
数据层（MyBatis-Flex + MySQL + InMemoryEmbeddingStore）
```
