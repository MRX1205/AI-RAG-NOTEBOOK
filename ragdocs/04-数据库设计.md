### 2.2 笔记本表 `notebook`（新增）

> 笔记本是用户组织知识的顶层单元。每个用户可以创建多个笔记本。

```sql
create table if not exists notebook
(
    id           bigint auto_increment comment 'id' primary key,
    userId       bigint                                 not null comment '创建用户id',
    title        varchar(256)                           not null comment '笔记本标题',
    description  varchar(1024)                          null comment '笔记本描述',
    sourceCount  int          default 0                 not null comment '来源数量（冗余字段）',
    coverImage   varchar(1024)                          null comment '封面图片URL',
    editTime     datetime     default CURRENT_TIMESTAMP not null comment '编辑时间',
    createTime   datetime     default CURRENT_TIMESTAMP not null comment '创建时间',
    updateTime   datetime     default CURRENT_TIMESTAMP not null on update CURRENT_TIMESTAMP comment '更新时间',
    isDelete     tinyint      default 0                 not null comment '是否删除',
    INDEX idx_userId (userId)
) comment '笔记本' collate = utf8mb4_unicode_ci;
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | bigint | 主键，雪花ID |
| `userId` | bigint | 关联用户表，标识归属 |
| `title` | varchar(256) | 笔记本标题，如"基于LangChain4j" |
| `description` | varchar(1024) | 笔记本描述信息 |
| `sourceCount` | int | 来源数量（冗余字段，查询列表时避免 JOIN） |
| `coverImage` | varchar(1024) | 笔记本封面图 URL（可选） |

### 2.3 知识来源表 `source`（新增）

> 来源是用户上传到笔记本中的文档。每条来源记录对应一个上传的文件或输入的文本。

```sql
create table if not exists source
(
    id            bigint auto_increment comment 'id' primary key,
    notebookId    bigint                                 not null comment '所属笔记本id',
    userId        bigint                                 not null comment '上传用户id',
    fileName      varchar(512)                           not null comment '文件名/来源名称',
    fileType      varchar(64)                            not null comment '文件类型：pdf/txt/md/docx/text',
    filePath      varchar(1024)                          null comment '文件存储路径（文件上传时有值）',
    fileSize      bigint       default 0                 not null comment '文件大小（字节）',
    content       mediumtext                             null comment '来源的纯文本内容（用于预览）',
    segmentCount  int          default 0                 not null comment '文本分块数量',
    status        varchar(32)  default 'processing'      not null comment '处理状态：processing/completed/failed',
    errorMessage  varchar(1024)                          null comment '处理失败的错误信息',
    createTime    datetime     default CURRENT_TIMESTAMP not null comment '创建时间',
    updateTime    datetime     default CURRENT_TIMESTAMP not null on update CURRENT_TIMESTAMP comment '更新时间',
    isDelete      tinyint      default 0                 not null comment '是否删除',
    INDEX idx_notebookId (notebookId),
    INDEX idx_userId (userId)
) comment '知识来源' collate = utf8mb4_unicode_ci;
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `fileType` | varchar(64) | 文件类型枚举：pdf, txt, md, docx, text（手动输入） |
| `filePath` | varchar(1024) | 文件在服务器上的存储路径 |
| `content` | mediumtext | 解析后的纯文本内容（用于页面预览，限制长度） |
| `segmentCount` | int | LangChain4j 文本分块后的数量 |
| `status` | varchar(32) | 处理状态，用于异步处理场景 |

### 2.5 测验表 `quiz`（新增）

> 存储用户生成的测验及其题目。

```sql
create table if not exists quiz
(
    id            bigint auto_increment comment 'id' primary key,
    notebookId    bigint                                 not null comment '所属笔记本id',
    userId        bigint                                 not null comment '生成用户id',
    title         varchar(512)                           not null comment '测验标题',
    questionCount int                                    not null comment '题目数量',
    difficulty    varchar(32)                            not null comment '难度：easy/medium/hard',
    questions     mediumtext                             not null comment '题目JSON数组',
    sourceIds     varchar(1024)                          null comment '参考的来源id列表（逗号分隔）',
    createTime    datetime     default CURRENT_TIMESTAMP not null comment '创建时间',
    updateTime    datetime     default CURRENT_TIMESTAMP not null on update CURRENT_TIMESTAMP comment '更新时间',
    isDelete      tinyint      default 0                 not null comment '是否删除',
    INDEX idx_notebookId (notebookId),
    INDEX idx_userId (userId)
) comment '测验' collate = utf8mb4_unicode_ci;
```

**`questions` 字段 JSON 结构**：

```json
[
  {
    "questionId": 1,
    "question": "题目文本",
    "options": [
      {"label": "A", "text": "选项文本"},
      {"label": "B", "text": "选项文本"},
      {"label": "C", "text": "选项文本"},
      {"label": "D", "text": "选项文本"}
    ],
    "correctAnswer": "B",
    "explanation": "答案解释"
  }
]
```

> **设计思路**：将测验题目以 JSON 格式存储在 `questions` 字段中，而不是拆成独立的题目表。原因：
> 1. 题目和测验是强绑定关系，不需要独立查询
> 2. JSON 格式更灵活，避免过多的表关联
> 3. MyBatis-Flex 可以用 JSON 类型处理器自动映射

## 4. 向量数据库说明

> 除了 MySQL 存储业务数据，RAG 系统还需要一个向量数据库来存储文档的向量索引。

### 4.1 开发阶段方案

使用 LangChain4j 内置的 **InMemoryEmbeddingStore**（内存向量存储）：

- ✅ 无需额外安装数据库
- ✅ 启动快速，适合开发调试
- ⚠️ 应用重启后数据丢失（可通过序列化到文件持久化）
- ⚠️ 不适合大规模数据

### 4.2 生产阶段方案（可选升级）

可升级到独立的向量数据库：

| 方案 | 说明 | 适合场景 |
|------|------|---------|
| InMemoryEmbeddingStore + 文件持久化 | 序列化到 JSON 文件 | 毕业设计演示 |
| Milvus | 开源向量数据库 | 生产环境 |
| Elasticsearch | 搜索引擎 + 向量检索 | 已有 ES 基础设施 |

> **毕业设计建议**：使用 InMemoryEmbeddingStore + JSON 文件持久化即可满足需求。每个笔记本维护一个独立的向量存储实例。
