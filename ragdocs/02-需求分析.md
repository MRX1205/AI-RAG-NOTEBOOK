# 需求分析文档

## 1. 功能需求

### 1.1 笔记本管理模块

> 笔记本（Notebook）是系统的顶层组织单元，类似于 NotebookLM 中的笔记本概念。每个笔记本是一个独立的知识空间。

#### 1.1.1 笔记本 CRUD

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 创建笔记本 | 用户创建新笔记本，输入名称和描述 | P0 |
| 查看笔记本列表 | 展示当前用户的所有笔记本，卡片式布局 | P0 |
| 编辑笔记本 | 修改笔记本名称和描述 | P1 |
| 删除笔记本 | 逻辑删除笔记本及其所有关联数据 | P1 |

#### 1.1.2 笔记本详情页

笔记本详情页采用 **三栏布局**（参考 NotebookLM 截图）：

```
┌──────────┬─────────────────────┬──────────┐
│  来源栏   │      对话主区域       │ Studio栏 │
│ (左侧栏)  │    (中间对话区)       │ (右侧栏)  │
│          │                     │          │
│ - 来源列表 │ - 对话消息流          │ - 报告列表 │
│ - 添加来源 │ - 输入框             │ - 测验列表 │
│ - 来源勾选 │ - 推荐问题           │ - 笔记列表 │
└──────────┴─────────────────────┴──────────┘
```

### 1.2 来源管理模块（知识库来源）

> 来源（Source）是 RAG 系统的知识数据入口。用户上传的每个文件或文本都被视为一个来源。

#### 1.2.1 来源上传

| 上传方式 | 说明 | 优先级 |
|---------|------|--------|
| 文件上传 | 支持 PDF、TXT、Markdown、Word（.docx）文件 | P0 |
| 复制粘贴文本 | 手动输入或粘贴文本内容作为来源 | P1 |
| 网页链接 | 输入 URL，系统自动抓取网页内容（可选功能） | P2 |

#### 1.2.2 来源处理流程

```
用户上传文件
    ↓
文件存储（本地磁盘 / OSS）
    ↓
文档解析（提取纯文本内容）
    ↓
文本分块（Text Splitting）
    ↓
向量化（Embedding）
    ↓
存入向量数据库（带元数据标记）
    ↓
来源记录写入 MySQL
```

#### 1.2.3 来源管理功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 来源列表展示 | 左侧栏展示笔记本下的来源列表，显示文件名、类型图标 | P0 |
| 来源选择/取消 | 勾选/取消勾选来源，控制对话和生成时参考的范围 | P0 |
| 删除来源 | 删除来源记录和向量数据库中对应的文档分块 | P1 |
| 来源详情预览 | 点击来源可查看原始内容摘要 | P2 |

### 1.3 AI 对话模块（RAG 增强对话）

> 核心功能：用户基于自己上传的知识来源与 AI 进行对话。AI 的回答必须严格基于文档内容，并标注引用。

#### 1.3.1 对话功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| RAG 对话 | 用户发送消息，系统检索相关文档片段 + LLM 生成回答 | P0 |
| 流式输出 | AI 回答以打字机效果逐字输出（SSE） | P0 |
| 引用标注 | 回答中对关键观点附带 [n] 引用角标 | P0 |
| 推荐问题 | 每次回答后自动生成 3 个推荐后续问题 | P1 |
| 对话历史 | 保存当前笔记本的对话记录 | P1 |
| 保存到笔记 | 将某条 AI 回答保存为笔记 | P2 |

#### 1.3.2 RAG 检索流程

```
用户输入问题
    ↓
查询向量化（Query Embedding）
    ↓
向量相似度检索（Top-K 相关文档片段）
    ↓
（可选）重排序（Re-ranking）
    ↓
组装 Prompt（系统提示 + 文档片段 + 用户问题）
    ↓
调用 LLM（DeepSeek）生成回答
    ↓
流式返回到前端
```

### 1.4 ⭐ 报告生成模块（Report）— 重点复刻

> 报告功能允许用户将知识库中的文档内容转化为结构化的报告文档。参考 NotebookLM 的 Studio 面板中的「报告」按钮。

#### 1.4.1 报告类型

| 报告类型 | 英文名 | 说明 |
|---------|--------|------|
| 简报文档 | Briefing Doc | 将来源内容概括为简洁的简报，提取关键信息 |
| 学习指南 | Study Guide | 按知识点层次结构整理内容，适合复习 |
| 常见问题 | FAQ | 从文档中提取潜在问题并生成问答对 |
| 时间线 | Timeline | 按时间或逻辑顺序整理事件或概念 |
| 自定义报告 | Custom | 用户输入自定义提示词，自由控制报告格式和内容 |

#### 1.4.2 报告生成流程

```
用户选择报告类型（或输入自定义要求）
    ↓
选择参考的来源（默认全部来源）
    ↓
系统检索所有选中来源的文档片段
    ↓
组装「报告生成 Prompt」
    ↓
调用 LLM 生成结构化报告
    ↓
流式输出报告内容
    ↓
保存到 Studio / 报告列表
```

#### 1.4.3 报告功能要求

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 选择报告类型 | 提供预设类型按钮 + 自定义输入框 | P0 |
| 生成报告 | 调用 LLM 基于文档生成 Markdown 格式报告 | P0 |
| 流式输出 | 报告内容流式逐步展示 | P0 |
| 报告列表 | 在 Studio 面板展示历史生成的报告 | P0 |
| 查看报告 | 点击查看报告的完整内容（Markdown 渲染） | P0 |
| 删除报告 | 删除已生成的报告 | P1 |
| 导出报告 | 将报告导出为 Markdown 或 PDF 文件 | P2 |

### 1.5 ⭐ 测验生成模块（Quiz）— 重点复刻

> 测验功能允许系统基于知识库文档自动生成选择题，用户可以互动答题、获得即时反馈。参考 NotebookLM 的 Studio 面板中的「测验」按钮。

#### 1.5.1 测验配置选项

| 配置项 | 选项 | 说明 |
|--------|------|------|
| 题目数量 | 5 / 10 / 15 / 20 | 控制生成的题目数量 |
| 难度级别 | 简单 / 中等 / 困难 | 控制题目的深度和复杂性 |
| 范围选择 | 全部来源 / 选中来源 | 控制出题的知识范围 |

#### 1.5.2 测验题目结构

每道测验题包含以下字段：

```json
{
  "questionId": 1,
  "question": "在 RAG 系统中，检索增强生成的核心步骤是什么？",
  "options": [
    { "label": "A", "text": "直接使用 LLM 回答用户问题" },
    { "label": "B", "text": "先检索相关文档，再结合文档内容让 LLM 生成回答" },
    { "label": "C", "text": "将所有文档一次性输入 LLM" },
    { "label": "D", "text": "使用规则引擎匹配关键词回答" }
  ],
  "correctAnswer": "B",
  "explanation": "RAG 的核心是先从知识库中检索与问题相关的文档片段，再将这些片段作为上下文提供给 LLM，让 LLM 基于检索到的内容生成回答。"
}
```

#### 1.5.3 测验交互流程

```
用户点击「生成测验」
    ↓
配置测验参数（数量、难度）
    ↓
系统检索选中来源的文档片段
    ↓
组装「测验生成 Prompt」
    ↓
LLM 返回 JSON 格式题目数组
    ↓
前端逐题展示
    ↓
用户选择答案 → 即时判定对错
    ↓
提供解释说明 + 来源引用
    ↓
全部答完 → 展示成绩概览
    ↓
保存测验记录
```

#### 1.5.4 测验功能要求

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 配置测验参数 | 数量、难度选择弹窗 | P0 |
| 生成测验题目 | 调用 LLM 基于文档生成选择题 JSON | P0 |
| 逐题展示 | 每次展示一题，用户选择后进入下一题 | P0 |
| 即时反馈 | 选择后立即显示对/错 + 正确答案 + 解释 | P0 |
| 成绩统计 | 答题完成后展示得分、正确率、用时 | P0 |
| 测验列表 | 在 Studio 面板展示历史测验列表 | P0 |
| 查看历史记录 | 查看之前的测验详情和答题情况 | P1 |
| 重新测验 | 基于相同配置重新生成一套测验 | P1 |

## 2. 非功能需求

### 2.1 性能需求

| 指标 | 要求 |
|------|------|
| 文件上传 | 单文件最大 20MB，支持并发上传 |
| 文档解析 | 单文件解析时间 < 30s |
| RAG 对话 | 首 token 响应时间 < 3s |
| 报告生成 | 流式输出，首 token < 5s |
| 测验生成 | 整体生成时间 < 15s |
| 页面加载 | 首屏加载 < 2s |

### 2.2 安全需求

| 指标 | 要求 |
|------|------|
| 用户鉴权 | 所有 API 需登录后访问（已有 Session 机制） |
| 数据隔离 | 用户只能访问/操作自己的笔记本和数据 |
| 密码安全 | 密码加密存储（已实现） |
| 文件安全 | 上传文件类型白名单校验 |

### 2.3 可用性需求

| 指标 | 要求 |
|------|------|
| 响应式设计 | 适配桌面端浏览器（1280px+） |
| 错误提示 | 友好的中文错误提示信息 |
| 加载状态 | 所有异步操作有加载动画/骨架屏 |
| 操作反馈 | 所有用户操作有 Toast 提示 |
