# 阶段四教程：报告生成 + 测验功能

> 本教程将教你实现两个 AI 驱动的功能：基于文档自动生成报告，以及基于文档生成选择题测验并自动评分。

---

## 1. 这个功能完成了什么

完成后，系统具备以下能力：
- ✅ 用户可以**一键生成 5 种类型的报告**（简报/学习指南/FAQ/时间线/自定义）
- ✅ 报告以**流式方式**实时显示生成过程
- ✅ 报告自动保存到数据库，可以随时查看历史报告
- ✅ 用户可以**生成选择题测验**，选择题数（5/10/15/20）和难度（简单/中等/困难）
- ✅ 测验支持**交互式答题**，提交后立即显示正确答案和解析
- ✅ 系统**自动评分**（百分制），记录答题历史

## 2. 为什么这么做

### 2.1 为什么要让 AI 生成结构化 JSON 题目？

测验题目需要有固定格式（题干、4 个选项、正确答案、解析）。我们在 Prompt 中要求 LLM 严格按 JSON 格式输出，这样前端可以直接解析和渲染。

```json
[
  {
    "questionId": 1,
    "question": "题目文本",
    "options": [
      {"label": "A", "text": "选项内容"},
      {"label": "B", "text": "选项内容"}
    ],
    "correctAnswer": "A",
    "explanation": "解析"
  }
]
```

### 2.2 为什么报告用 SSE 而测验不用？

- **报告**：内容长（几百到几千字），生成慢，用 SSE 让用户看到生成过程
- **测验**：LLM 需要完整生成 JSON 后才能解析，不适合边流边解析

#### 3.1.2 报告生成接口（SSE 流式）

```java
@Slf4j
@RestController
@RequestMapping("/report")
public class ReportController {

    @Resource
    private ReportService reportService;
    @Resource
    private RagService ragService;
    @Resource
    private StreamingChatModel streamingChatModel;

    /**
     * 生成报告（SSE 流式输出）
     * 
     * 完整流程：
     * 1. 检索文档片段作为素材
     * 2. 根据报告类型选择对应的 Prompt 模板
     * 3. SSE 流式调用 LLM 生成报告
     * 4. 生成完毕后自动保存到数据库
     */
    @PostMapping(value = "/generate", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public SseEmitter generateReport(
            @RequestBody ReportGenerateRequest generateRequest,
            HttpServletRequest request) {

        // 参数 + 权限校验（省略）...

        // 1. 检索文档内容作为报告素材
        String context = ragService.retrieveFormattedContext(
                notebookId, "全面总结所有文档要点", generateRequest.getSourceIds());

        // 2. 根据报告类型构建对应的 Prompt
        String prompt = buildReportPrompt(
                generateRequest.getReportType(),
                generateRequest.getCustomPrompt(),
                context);

        // 3. SSE 流式生成
        SseEmitter emitter = new SseEmitter(180000L);
        StringBuilder fullContent = new StringBuilder();  // 积累完整内容

        new Thread(() -> {
            try {
                var usrMsg = UserMessage.from(prompt);
                
                streamingChatModel.chat(List.of(usrMsg),
                    new StreamingChatResponseHandler() {
                        @Override
                        public void onPartialResponse(String partialResponse) {
                            if (partialResponse != null && !partialResponse.isEmpty()) {
                                fullContent.append(partialResponse);  // 累积
                                try {
                                    emitter.send(SseEmitter.event().data(partialResponse));
                                } catch (IOException e) {
                                    log.error("SSE 发送失败", e);
                                }
                            }
                        }

                        @Override
                        public void onCompleteResponse(ChatResponse completeResponse) {
                            // 4. 生成完毕，保存到数据库
                            try {
                                Report report = Report.builder()
                                        .notebookId(notebookId)
                                        .userId(loginUser.getId())
                                        .title(generateReportTitle(generateRequest.getReportType()))
                                        .reportType(generateRequest.getReportType())
                                        .content(fullContent.toString())
                                        .build();
                                reportService.save(report);
                                
                                // 发送完成信号（包含报告 ID）
                                emitter.send(SseEmitter.event()
                                        .data("[DONE:" + report.getId() + "]"));
                                emitter.complete();
                            } catch (Exception e) {
                                log.error("保存报告失败", e);
                            }
                        }

                        @Override
                        public void onError(Throwable error) {
                            // 错误处理
                        }
                    });
            } catch (Exception e) {
                // 异常处理
            }
        }).start();

        return emitter;
    }
```

#### 3.1.3 报告类型 Prompt 模板

**这是 Prompt 工程的核心**——不同类型的报告需要不同的指令：

```java
    private String buildReportPrompt(String reportType, String customPrompt, String context) {
        String typeInstruction = switch (reportType) {
            
            case "briefing" -> """
                请基于以下文档内容，生成一份专业的简报文档。要求：
                1. 包含执行摘要
                2. 列出核心要点（每个要点配有详细说明）
                3. 给出结论和建议
                使用 Markdown 格式排版。""";
            
            case "study_guide" -> """
                请基于以下文档内容，生成一份全面的学习指南。要求：
                1. 概述学习目标
                2. 按主题/章节组织知识点
                3. 每个知识点给出关键定义和解释
                4. 在末尾添加重点回顾清单
                使用 Markdown 格式排版。""";
            
            case "faq" -> """
                请基于以下文档内容，生成一份 FAQ 文档。要求：
                1. 提取文档中最重要的 8-12 个问题
                2. 每个问题给出简洁准确的回答
                3. 回答直接引用文档原文
                使用 Markdown 格式排版。""";
            
            case "timeline" -> """
                请基于以下文档内容，生成一份时间线文档。要求：
                1. 按时间顺序梳理关键事件
                2. 每个事件标注时间、描述和影响
                使用 Markdown 格式排版。""";
            
            case "custom" -> 
                "请根据以下指令生成报告：\n" + 
                (customPrompt != null ? customPrompt : "请总结文档主要内容");
            
            default -> "请总结以下文档内容。";
        };
        
        return typeInstruction + "\n\n文档内容：\n" + context;
    }

### 3.2 测验后端（QuizController.java）

#### 3.2.1 测验生成

```java
@Slf4j
@RestController
@RequestMapping("/quiz")
public class QuizController {

    @Resource
    private QuizService quizService;
    @Resource
    private RagService ragService;
    @Resource
    private ChatModel chatModel;  // 非流式模型（需要完整 JSON）

    /**
     * 生成测验
     * 
     * 流程：
     * 1. 检索文档内容
     * 2. 构建 Prompt 要求 LLM 生成 JSON 格式选择题
     * 3. 从 LLM 响应中提取 JSON
     * 4. 保存到数据库
     */
    @PostMapping("/generate")
    public BaseResponse<QuizVO> generateQuiz(
            @RequestBody QuizGenerateRequest generateRequest,
            HttpServletRequest request) {

        // 参数校验 + 权限校验...

        // 1. 检索文档
        String context = ragService.retrieveFormattedContext(
                notebookId, "全部内容要点", generateRequest.getSourceIds());

        // 2. 构建测验 Prompt
        String prompt = buildQuizPrompt(
                generateRequest.getQuestionCount(),
                generateRequest.getDifficulty(),
                context);

        // 3. 调用 LLM（非流式，需要完整 JSON）
        try {
            ChatResponse response = chatModel.chat(
                    ChatRequest.builder()
                            .messages(List.of(UserMessage.from(prompt)))
                            .build());
            
            // 4. 从 LLM 响应中提取 JSON
            //    LLM 可能返回 ```json ... ``` 包裹的格式
            String questionsJson = extractJson(response.aiMessage().text());
            
            // 5. 验证 JSON 格式
            JSONArray jsonArray = JSONUtil.parseArray(questionsJson);
            if (jsonArray.isEmpty()) {
                throw new BusinessException(ErrorCode.SYSTEM_ERROR, "生成的题目为空");
            }

            // 6. 保存到数据库
            Quiz quiz = Quiz.builder()
                    .notebookId(notebookId)
                    .userId(loginUser.getId())
                    .title("测验 - " + getDifficultyText(generateRequest.getDifficulty()))
                    .questionCount(jsonArray.size())
                    .difficulty(generateRequest.getDifficulty())
                    .questions(questionsJson)  // JSON 字符串存入 TEXT 列
                    .build();
            quizService.save(quiz);

            return ResultUtils.success(quizService.getQuizVO(quiz));
        } catch (Exception e) {
            throw new BusinessException(ErrorCode.SYSTEM_ERROR, "测验生成失败");
        }
    }
```

#### 3.2.2 从 LLM 响应中提取 JSON

LLM 有时会返回额外的解释文字，我们需要提取出纯 JSON 部分：

```java
    /**
     * 从 LLM 响应中提取 JSON 数组
     * 
     * LLM 可能的输出格式：
     * 1. ```json\n[...]\n```        ← 常见格式
     * 2. 一些解释文字\n[...]\n      ← 有时会带前缀
     * 3. [...]                      ← 最简格式
     */
    private String extractJson(String text) {
        // 情况 1：尝试找 ```json ... ``` 包裹
        int start = text.indexOf("```json");
        if (start != -1) {
            start = text.indexOf("\n", start) + 1;  // 跳过 ```json 那一行
            int end = text.indexOf("```", start);
            if (end != -1) {
                return text.substring(start, end).trim();
            }
        }
        // 情况 2 和 3：尝试找 [ ... ]
        start = text.indexOf("[");
        int end = text.lastIndexOf("]");
        if (start != -1 && end > start) {
            return text.substring(start, end + 1);
        }
        return text;  // 兜底
    }
```

#### 3.2.3 测验 Prompt

```java
    private String buildQuizPrompt(int questionCount, String difficulty, String context) {
        String difficultyDesc = switch (difficulty) {
            case "easy" -> "简单（基础概念和事实性问题）";
            case "medium" -> "中等（需要理解和分析的问题）";
            case "hard" -> "困难（需要综合分析和深度理解的问题）";

        return """
            请根据以下文档内容，生成 %d 道选择题。
            
            要求：
            1. 难度级别：%s
            2. 每道题有且仅有 4 个选项（A/B/C/D）
            3. 必须有且仅有 1 个正确答案
            4. 提供答案解释
            5. 严格基于文档内容出题
            
            请严格按照以下 JSON 数组格式输出：
            ```json
            [
              {
                "questionId": 1,
                "question": "题目文本",
                "options": [
                  {"label": "A", "text": "选项A"},
                  {"label": "B", "text": "选项B"},
                  {"label": "C", "text": "选项C"},
                  {"label": "D", "text": "选项D"}

                "explanation": "解析说明"

            文档内容：
            %s
            """.formatted(questionCount, difficultyDesc, context);

#### 3.2.4 自动评分

```java
    /**
     * 提交测验答案并自动评分
     */
    @PostMapping("/submit")
    public BaseResponse<QuizRecord> submitQuiz(
            @RequestBody QuizSubmitRequest submitRequest,
            HttpServletRequest request) {

        User loginUser = userService.getLoginUser(request);
        Quiz quiz = quizService.getById(submitRequest.getQuizId());

        // 解析正确答案
        JSONArray questions = JSONUtil.parseArray(quiz.getQuestions());
        int correctCount = 0;
        int totalCount = questions.size();

        // 逐题对比答案
        for (QuizSubmitRequest.QuizAnswer answer : submitRequest.getAnswers()) {
            if (answer.getQuestionId() != null && answer.getQuestionId() <= totalCount) {
                var question = questions.getJSONObject(answer.getQuestionId() - 1);
                String correct = question.getStr("correctAnswer");
                // 不区分大小写对比
                if (correct != null && correct.equalsIgnoreCase(answer.getAnswer())) {
                    correctCount++;
                }
            }
        }

        // 计算百分制得分
        int score = totalCount > 0 
                ? (int) Math.round((double) correctCount / totalCount * 100) 
                : 0;

        // 保存答题记录
        QuizRecord record = QuizRecord.builder()
                .quizId(submitRequest.getQuizId())
                .userId(loginUser.getId())
                .score(score)
                .correctCount(correctCount)
                .totalCount(totalCount)
                .timeCost(submitRequest.getTimeCost())
                .answers(JSONUtil.toJsonStr(submitRequest.getAnswers()))
                .build();
        quizRecordMapper.insert(record);

        return ResultUtils.success(record);
    }
```

#### 3.3.1 StudioPanel.vue 核心功能

Studio 面板分为**报告区**和**测验区**，核心交互：

**报告生成流程**：
```typescript
/** 使用 fetch + ReadableStream 接收 SSE（POST 请求） */
const handleGenerateReport = async () => {
  const response = await fetch('/api/report/generate', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      notebookId: props.notebookId,
      reportType: selectedReportType.value,
      sourceIds: props.selectedSourceIds,
    }),
    credentials: 'include',
  })

  const reader = response.body!.getReader()
  const decoder = new TextDecoder()
  let reportContent = ''

  // 实时展示报告内容
  currentReport.value = { title: '生成中...', content: '' }
  reportViewerVisible.value = true

  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    const text = decoder.decode(value, { stream: true })
    
    // 解析 SSE 格式的数据
    const lines = text.split('\n')
    for (const line of lines) {
      if (line.startsWith('data:')) {
        const data = line.substring(5).trim()
        if (data.startsWith('[DONE')) break
        reportContent += data
        currentReport.value.content = reportContent
      }
    }
  }
}
```

**注意**：因为报告生成使用 POST 请求（需要传 body），不能用 `EventSource`（只支持 GET），所以这里用 `fetch` + `ReadableStream` 手动解析 SSE 数据。

## 小结

本阶段扩展了 RAG 的应用场景，从"对话"拓展到"报告生成"和"测验"：

**报告生成**：
```
用户选择报告类型（如"学习指南"）
    ↓
RagService 检索文档片段
    ↓
根据类型选择对应 Prompt 模板
    ↓
StreamingChatModel 流式生成 Markdown 报告
    ↓ 前端实时显示
    ↓
生成完毕 → 自动保存到数据库
```

**测验生成**：
```
用户选择题数(10)和难度(中等)
    ↓
RagService 检索文档片段
    ↓
构建 Prompt 要求输出 JSON 格式选择题
    ↓
ChatModel 一次性生成完整 JSON
    ↓
解析 JSON → 保存到数据库
    ↓
前端渲染交互式答题界面
    ↓
用户答题 → 提交 → 自动评分
```

**至此，RAG 知识库系统的核心功能全部完成！** 🎉

系统架构回顾：
- **阶段一**：基础设施（用户认证、笔记本管理）
- **阶段二**：RAG 数据入库（上传 → 解析 → 分块 → 向量化 → 存储）
- **阶段三**：RAG 对话（检索 → 上下文注入 → 流式回答）  
- **阶段四**：报告 + 测验（检索 → Prompt 工程 → 结构化输出）
