# 阶段报告：任务一 - 个人信息修改功能

## 完成时间
2026-03-06

## 任务目标
实现用户个人信息修改功能，包括：修改用户名、修改密码、上传头像，并确保修改后导航栏实时同步更新。

---

## 实现内容

### 1. 后端接口（已有，验证可用）

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 修改用户名 | PUT | `/user/update/username` | 需登录，更新 `userName` 字段 |
| 修改密码 | PUT | `/user/update/password` | 需登录，校验旧密码后更新 |
| 上传头像 | POST | `/user/upload/avatar` | 需登录，保存到 `uploads/avatars/`，返回相对 URL |

所有接口均已在 `UserController.java` 中实现，`UserServiceImpl.java` 包含完整业务逻辑：
- 修改密码时校验旧密码与数据库中 MD5 哈希值匹配
- 头像文件保存使用 `Files.copy + InputStream` 规避 Tomcat 相对路径问题
- 头像大小限制 2MB，格式限制 jpg/png/gif/webp

### 2. 前端页面（UserProfilePage.vue）

页面分三个卡片：

**头像区域**
- 点击头像触发文件选择，本地预览后上传
- 上传成功后调用 `loginUserStore.setLoginUser()` 同步更新导航栏头像

**修改用户名**
- 输入框 + 保存按钮
- 成功后 `setLoginUser()` 同步更新导航栏用户名

**修改密码**
- 旧密码 + 新密码 + 确认新密码三个输入框
- 前端校验：非空、长度 6-20、两次一致
- 修改成功后提示重新登录，自动跳转登录页

### 3. 响应式状态修复（核心修复）

**问题**：原实现使用 `ref<LoginUserVO>` 存储登录状态，`setLoginUser()` 替换整个对象时，
已捕获的组件本地引用变为过时对象，导致用户名/头像修改后导航栏无法实时更新。

**修复方案**：将 `loginUser` 从 `ref<LoginUserVO>` 改为 `reactive<LoginUserVO>`，
`setLoginUser()` 改为 `Object.assign` 原地更新：

```ts
// stores/loginUser.ts
const loginUser = reactive<API.LoginUserVO>({ userName: '未登录' })

function setLoginUser(newLoginUser: API.LoginUserVO | { userName: string }) {
  // 先清空旧属性，再合并新数据
  Object.keys(loginUser).forEach((k) => delete (loginUser as Record<string, unknown>)[k])
  Object.assign(loginUser, newLoginUser)
}
```

**效果**：所有通过 `loginUserStore.loginUser` 访问的组件（HomePage、AdminLayout、UserProfilePage 等）
共享同一个响应式代理对象，`setLoginUser` 修改后立即同步到所有使用处。

### 4. 导航栏集成

`HomePage.vue` 导航栏中：
- 头像：`:src="loginUser.userAvatar ? '/api' + loginUser.userAvatar : undefined"`
- 用户名：`{{ userDisplayName }}`（computed，来自 reactive loginUser）
- 修改后自动响应变化，无需手动刷新
