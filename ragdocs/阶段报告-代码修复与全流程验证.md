## 二、关键修复详解

### Fix-1：RagService 来源过滤（最重要）

**参考示例：** `langchain4j-examples-main/rag-examples/src/main/java/_3_advanced/_05_Advanced_RAG_with_Metadata_Filtering_Examples.java`

**问题本质：**  
原代码先检索 top-5（所有来源混合），再手动过滤。若用户只勾选了 2 个来源，但 top-5 全来自未勾选的来源，过滤后结果为空，AI 无法回答。

**修复前：**
```java
ContentRetriever retriever = builder.build();
List<Content> contents = retriever.retrieve(Query.from(query));
// 检索后再手动过滤 ← 问题所在
if (sourceIds != null && !sourceIds.isEmpty()) {
    results = contents.stream().filter(content -> sourceIdStrings.contains(...)).collect(...);
}
```

**修复后：**
```java
// 参考 _05_Advanced_RAG_with_Metadata_Filtering_Examples.java
if (sourceIds != null && !sourceIds.isEmpty()) {
    Filter sourceFilter = metadataKey("sourceId").isIn(sourceIdStrings);
    builder.filter(sourceFilter);  // 检索时过滤 ← 正确做法
}
ContentRetriever retriever = builder.build();
List<Content> contents = retriever.retrieve(Query.from(query));
```

### Fix-5：EmbeddingConfig SSL 修复（影响最广）

**问题发现过程：**
1. curl 调用 DashScope API → ✅ 正常（HTTP 200，返回向量）
2. Java (OkHttp) 调用 DashScope API → ❌ `SSLHandshakeException: Remote host terminated the handshake`
3. 诊断：系统启用了 `AppsConnect_Mac` VPN，该 VPN 做 SSL 检查，OkHttp 不信任其 CA
4. 尝试 `KeychainStore` → 错误变为 `PKIX path building failed`（更明确的证书不信任错误）

**根因：** `QwenEmbeddingModel` 使用 `dashscope-sdk-java 2.20.6`，底层 OkHttp 使用自己的 TrustManager，不受 Java 系统属性影响。

**修复方案：** DashScope 提供 OpenAI 兼容接口（`https://dashscope.aliyuncs.com/compatible-mode/v1`），改用 `OpenAiEmbeddingModel`。该模块使用 Spring RestClient，与 DeepSeek 聊天模型共用同一 HTTP 客户端（已验证可用）。

```java
// 修复前：
return QwenEmbeddingModel.builder().apiKey(apiKey).modelName(modelName).build();

// 修复后：
return OpenAiEmbeddingModel.builder()
        .baseUrl("https://dashscope.aliyuncs.com/compatible-mode/v1")
        .apiKey(apiKey)
        .modelName(modelName)
        .build();
```

## 五、遇到的问题及解决方案

| 问题 | 根因 | 解决方案 |
|------|------|---------|
| DashScope Embedding SSL 握手失败 | VPN（AppsConnect_Mac）做 SSL 拦截，OkHttp 不信任其 CA | 改用 DashScope OpenAI 兼容接口 + Spring RestClient |
| pom.xml 版本号统一失败 | Maven Central 网络不通，无法下载新 POM | 使用离线模式（`-o`）+ Properties 统一已缓存版本 |
| `new Thread()` 每次对话创建新线程 | 未使用线程池 | 改为 `Executors.newCachedThreadPool()` |
