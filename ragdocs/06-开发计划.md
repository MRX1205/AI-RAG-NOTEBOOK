# 开发计划

## 1. 开发阶段划分

整个项目按照 **功能递进、依赖前置** 的原则，划分为 **5 个开发阶段**：

```
阶段一        阶段二        阶段三       阶段四        阶段五
基础设施  →  知识库核心  →  AI 对话   →  报告+测验  →  优化完善
 (2天)       (3天)        (2天)       (4天)        (2天)
```

## 4. 技术攻关要点

### 4.1 RAG 检索质量

| 难点 | 解决方案 |
|------|---------|
| 文档分块粒度 | 使用 `DocumentSplitters.recursive(500, 50)` 递归分块 |
| 检索精度 | 调整 `minScore` 阈值 + 增加 `maxResults` |
| 引用准确性 | 在 Prompt 中强制要求 [n] 格式引用 |

### 4.2 LLM 输出格式控制

| 难点 | 解决方案 |
|------|---------|
| 测验 JSON 格式 | 在 Prompt 中提供 JSON Schema 示例 |
| JSON 解析失败 | 后端兜底：用正则/手动提取 JSON 数组 |
| 报告 Markdown 格式 | Prompt 明确要求 Markdown 标题格式 |

### 4.3 流式传输（SSE）

| 难点 | 解决方案 |
|------|---------|
| 后端 SSE 实现 | 使用 `SseEmitter` 或 `Flux<String>` |
| 前端 SSE 接收 | 使用 `EventSource` API |
| 跨域问题 | CorsConfig 已配置，注意 SSE 的 CORS |
| 断线重连 | EventSource 自带重连，设置合理 timeout |

### 4.4 Embedding 模型选择

| 方案 | 说明 | 推荐度 |
|------|------|--------|
| DeepSeek Embedding | 与现有 API key 复用 | ⭐⭐⭐ |
| 本地 Embedding（ONNX） | 无需 API，离线使用 | ⭐⭐ |
| OpenAI Embedding | 需另外 API | ⭐ |

> **注意**：当前 `application-local.yml` 中的 embedding-model 配置用的是 `deepseek-chat`，这是一个聊天模型而非 Embedding 模型。需要确认 DeepSeek 是否提供 Embedding API，或更换为其他 Embedding 服务。

## 6. 学习路线建议

### 6.1 LangChain4j 学习顺序

```
1. 阅读本项目现有代码
   └── AiNotebookService.java（理解 @SystemMessage 注解）
   └── AiNotebookServiceFactory.java（理解 AiServices 构建器）

2. LangChain4j 官方示例（本项目 langchain4j-examples-main/）
   └── rag-examples/Easy_RAG_Example.java（入门 RAG）
   └── rag-examples/Naive_RAG_Example.java（标准 RAG）
   └── rag-examples/_3_advanced/（进阶 RAG 技巧）

3. LangChain4j 官方文档
   └── https://docs.langchain4j.dev/

4. 实战：按阶段逐步实现
```

### 6.2 关键概念速查

| 概念 | 说明 | 对应类 |
|------|------|--------|
| Document | 一个完整文档 | `dev.langchain4j.data.document.Document` |
| TextSegment | 文档分块后的文本片段 | `dev.langchain4j.data.segment.TextSegment` |
| Embedding | 文本的向量表示 | `dev.langchain4j.data.embedding.Embedding` |
| EmbeddingStore | 向量存储 | `dev.langchain4j.store.embedding.EmbeddingStore` |
| EmbeddingModel | 向量化模型 | `dev.langchain4j.model.embedding.EmbeddingModel` |
| ContentRetriever | 内容检索器 | `dev.langchain4j.rag.content.retriever.ContentRetriever` |
| AiServices | AI 服务构建器 | `dev.langchain4j.service.AiServices` |
| @SystemMessage | 系统提示注解 | `dev.langchain4j.service.SystemMessage` |
