# 阶段报告：任务二 - 首页布局重构

## 完成时间
2026-03-06

## 任务目标
重构首页布局，使导航栏根据登录状态自适应显示，并将主体内容分为「精选知识库」和「我的笔记本」两个清晰区域。

---

## 实现内容

### 1. 导航栏重构（nav-links + nav-right）

**未登录状态：**
```
Logo | 首页 · 发现精选 | [注册] [登录]
```

**已登录普通用户：**
```
Logo | 首页 · 发现精选 | [⊞☰] [新建笔记本] [头像 用户名 ▼]
```

**已登录管理员：**
```
Logo | 首页 · 发现精选 | [⊞☰] [🔧管理后台] [新建笔记本] [头像 用户名 ▼]
```

**改动点：**
- 将「全部/我的笔记本/精选笔记本」Tab 过滤器替换为两个固定导航链接（首页 + 发现精选）
- 「发现精选」链接指向 `/explore` 页面
- 用户名显示在头像旁边（之前只显示头像）
- 下拉菜单保留「我的笔记」「个人中心」「退出登录」

### 2. 精选知识库区域（区域一）

- 标题「✨ 精选知识库」，右侧有「查看全部 →」链接指向 /explore
- 精选卡片横向滚动（`featured-scroll`）
- 卡片高度扩大至 195px，底部增加操作按钮区域
- **已登录用户**：卡片底部显示白色半透明「添加到我的笔记本」按钮
- **未登录用户**：卡片底部显示「登录后可添加到笔记本」灰色按钮，点击跳转登录
- 点击卡片（非按钮区域）进入 `/explore/:id` 只读详情页

### 3. 我的笔记本区域（区域二）

- 仅登录用户可见
- 标题「📚 我的笔记本」+ 右侧「新建」按钮
- 支持网格/列表双视图切换（⊞/☰）
- 空状态显示「还没有笔记本，点击新建」
- 第一个网格卡片始终是「新建笔记本」虚线卡片

### 4. 精选卡片克隆逻辑

```ts
// 防重复点击：使用 Set 记录正在克隆的笔记本 ID
const cloningIds = ref<Set<string | number>>(new Set())

const handleAddToMine = async (notebookId, event) => {
  event.stopPropagation()  // 阻止事件冒泡，避免触发进入详情
  cloningIds.value.add(notebookId)
  const res = await cloneNotebook(notebookId)
  if (res.data.code === 0) {
    message.success('已添加到我的笔记本！')
    await loadNotebookList()  // 刷新我的笔记本列表
  }
  cloningIds.value.delete(notebookId)
}
```
