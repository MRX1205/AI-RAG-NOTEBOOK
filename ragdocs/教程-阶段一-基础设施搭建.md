## Step 2 — 创建数据库表

### 做什么？

创建 5 张新表，分别支持：笔记本、来源、报告、测验、答题记录。

### 新建文件

📄 **sql/notebook.sql**

执行这个 SQL 文件即可创建所有表。本阶段重点是 `notebook` 表：

```sql
create table if not exists notebook
(
    id           bigint auto_increment comment 'id' primary key,
    userId       bigint                                 not null comment '创建用户id',
    title        varchar(256)                           not null comment '笔记本标题',
    description  varchar(1024)                          null comment '笔记本描述',
    sourceCount  int          default 0                 not null comment '来源数量',
    coverImage   varchar(1024)                          null comment '封面图片URL',
    editTime     datetime     default CURRENT_TIMESTAMP not null comment '编辑时间',
    createTime   datetime     default CURRENT_TIMESTAMP not null comment '创建时间',
    updateTime   datetime     default CURRENT_TIMESTAMP not null
                 on update CURRENT_TIMESTAMP comment '更新时间',
    isDelete     tinyint      default 0                 not null comment '是否删除',
    INDEX idx_userId (userId)
) comment '笔记本' collate = utf8mb4_unicode_ci;
```

### 关键设计解释

- **`sourceCount`**：这是一个**冗余字段**。按理说来源数量可以通过 `SELECT COUNT(*) FROM source WHERE notebookId = ?` 计算出来，但在笔记本列表页中，如果每个笔记本都要 JOIN 查一次，性能会很差。所以我们在 notebook 表中冗余存一份，每次增删来源时同步更新。
- **`isDelete`**：逻辑删除字段。删除记录时不真正删除数据（物理删除），而是把这个字段设为 1。MyBatis-Flex 的 `@Column(isLogicDelete = true)` 注解会自动处理：查询时自动加 `WHERE isDelete = 0`，删除时自动改为 `UPDATE ... SET isDelete = 1`。
- **`INDEX idx_userId`**：为 userId 建索引，因为查询笔记本列表时都要按 userId 过滤，索引能大幅提高查询速度。

### 操作

在 MySQL 客户端中运行 SQL：
```bash
mysql -u root -p lyhm_ai_rag < sql/notebook.sql
```

## Step 3 — 创建后端实体类（Entity）

### 做什么？

创建 `Notebook.java`，它是数据库 `notebook` 表在 Java 中的映射。

### 新建文件

📄 **src/main/java/com/lyhm/airag/model/entity/Notebook.java**

### 核心代码解析

```java
@Data                    // Lombok：自动生成 getter/setter/toString/equals 等
@Builder                 // Lombok：可以用链式调用创建对象
@NoArgsConstructor       // Lombok：无参构造函数
@AllArgsConstructor      // Lombok：全参构造函数
@Table("notebook")       // MyBatis-Flex：指定对应的数据库表名
public class Notebook implements Serializable {

    // 主键，使用雪花算法生成（分布式唯一ID，比自增ID更安全）
    @Id(keyType = KeyType.Generator, value = KeyGenerators.snowFlakeId)
    private Long id;

    // @Column 注解指定对应的数据库列名
    // MyBatis-Flex 默认是驼峰转下划线，但因为我们的列名就是驼峰
    // 所以需要显式指定
    @Column("userId")
    private Long userId;

    // 逻辑删除注解：查询时自动过滤 isDelete=1 的记录
    @Column(value = "isDelete", isLogicDelete = true)
    private Integer isDelete;
}
```

### 为什么要实现 Serializable？

Java 的序列化接口。当对象需要在网络传输（如存入 Session）或缓存中使用时，需要实现此接口。`serialVersionUID` 用于版本控制，防止反序列化时版本不匹配。

## Step 4 — 创建 DTO 和 VO

### 概念解释

Spring Boot 项目中常用三种对象来传递数据：

```
前端 ──→ DTO（请求参数） ──→ Controller ──→ Entity（数据库对象） ──→ VO（返回数据） ──→ 前端
```

| 类型 | 全称 | 用途 | 例子 |
|------|------|------|------|
| **Entity** | 实体类 | 对应数据库表 | Notebook.java |
| **DTO** | Data Transfer Object | 接收前端请求参数 | NotebookAddRequest.java |
| **VO** | View Object | 返回给前端的数据 | NotebookVO.java |

### 为什么不直接用 Entity？

1. **安全**：Entity 包含所有字段（如 isDelete、userId），直接暴露给前端不安全
2. **灵活**：DTO 只包含前端需要传的字段，VO 只包含前端需要展示的字段
3. **解耦**：数据库结构变化不会直接影响接口

### 新建文件

📄 **model/dto/notebook/NotebookAddRequest.java** — 创建请求（title, description）
📄 **model/dto/notebook/NotebookUpdateRequest.java** — 更新请求（id, title, description）
📄 **model/dto/notebook/NotebookQueryRequest.java** — 查询请求（继承 PageRequest，支持分页）
📄 **model/vo/NotebookVO.java** — 视图对象（id, title, description, sourceCount, createTime...）

## Step 5 — 创建 Mapper 接口

### 做什么？

Mapper 是 MyBatis-Flex 与数据库交互的接口层。

### 新建文件

📄 **mapper/NotebookMapper.java**

```java
public interface NotebookMapper extends BaseMapper<Notebook> {
    // 继承 BaseMapper 后自动获得：
    // - insert(entity)       插入
    // - deleteById(id)       删除
    // - updateById(entity)   更新
    // - selectOneById(id)    按ID查询
    // - selectAll()          查询所有
    // 不需要写任何方法，框架自动生成！
}
```

### 为什么不用写实现？

MyBatis-Flex 的 `BaseMapper<T>` 使用了**动态代理**技术。在运行时，框架会根据泛型参数 `T`（即 Notebook）自动生成 SQL 语句。比如调用 `selectOneById(1L)`，框架会自动生成 `SELECT * FROM notebook WHERE id = 1 AND isDelete = 0`。

## Step 6 — 创建 Service 层

### 做什么？

Service 层封装业务逻辑，是 Controller 和 Mapper 之间的桥梁。

### 新建文件

📄 **service/NotebookService.java** — 接口
📄 **service/impl/NotebookServiceImpl.java** — 实现

### 核心代码解析

```java
// 接口：定义业务契约
public interface NotebookService extends IService<Notebook> {
    NotebookVO getNotebookVO(Notebook notebook);      // Entity → VO 转换
    List<NotebookVO> getNotebookVOList(List<Notebook> list);  // 批量转换
}

// 实现类
@Service  // 注册为 Spring Bean
public class NotebookServiceImpl
        extends ServiceImpl<NotebookMapper, Notebook>  // 自动注入 Mapper
        implements NotebookService {

    @Override
    public NotebookVO getNotebookVO(Notebook notebook) {
        NotebookVO vo = new NotebookVO();
        // Hutool 的属性拷贝工具：将同名属性从 notebook 拷贝到 vo
        // 因为 NotebookVO 不包含 isDelete、userId，所以自动过滤了
        BeanUtil.copyProperties(notebook, vo);
        return vo;
    }
}
```

### 为什么 Service 要分接口和实现？

这是**面向接口编程**的体现（Java 设计原则之一）：
1. Controller 依赖接口，不依赖具体实现，方便替换
2. 单元测试时可以 Mock 接口
3. 接口定义清晰的业务契约，代码可读性更好

## Step 7 — 创建 Controller 层

### 做什么？

Controller 接收前端 HTTP 请求，调用 Service 处理业务，返回响应。

### 新建文件

📄 **controller/NotebookController.java** — 5 个 CRUD 接口

### 接口清单

| 方法 | 路径 | 功能 | 请求参数 | 返回值 |
|------|------|------|---------|--------|
| POST | `/notebook/add` | 创建笔记本 | NotebookAddRequest | Long（新ID） |
| GET | `/notebook/list` | 获取笔记本列表 | 无 | List\<NotebookVO\> |
| GET | `/notebook/get` | 获取笔记本详情 | id | NotebookVO |
| POST | `/notebook/update` | 更新笔记本 | NotebookUpdateRequest | Boolean |
| POST | `/notebook/delete` | 删除笔记本 | DeleteRequest | Boolean |

### 核心代码解析 — 以创建接口为例

```java
@PostMapping("/add")
public BaseResponse<Long> addNotebook(
        @RequestBody NotebookAddRequest notebookAddRequest,  // 从请求体解析JSON
        HttpServletRequest request) {                         // 获取HTTP请求信息

    // 1. 参数校验
    ThrowUtils.throwIf(notebookAddRequest == null, ErrorCode.PARAMS_ERROR);

    // 2. 获取当前登录用户（核心！每个接口都要做）
    //    如果用户未登录，getLoginUser 会抛出 NOT_LOGIN_ERROR 异常
    User loginUser = userService.getLoginUser(request);

    // 3. DTO → Entity 转换
    Notebook notebook = new Notebook();
    BeanUtil.copyProperties(notebookAddRequest, notebook);
    notebook.setUserId(loginUser.getId());  // 关联当前用户
    notebook.setSourceCount(0);              // 初始来源数为0

    // 4. 保存到数据库
    boolean result = notebookService.save(notebook);

    // 5. 返回新创建的笔记本ID
    return ResultUtils.success(notebook.getId());
}
```

### 数据隔离的实现

每个接口都会校验「当前登录用户」只能操作自己的笔记本：

```java
// 查询列表时：只查自己的
QueryWrapper queryWrapper = QueryWrapper.create()
        .eq("userId", loginUser.getId());

// 详情/更新/删除时：校验归属权
ThrowUtils.throwIf(!notebook.getUserId().equals(loginUser.getId()),
        ErrorCode.NO_AUTH_ERROR, "无权访问该笔记本");
```

## Step 9 — 前端首页改造

### 做什么？

将原来空白的 HomePage 改造为笔记本卡片列表页。

### 修改文件

📄 **src/pages/HomePage.vue**

### 核心代码解析 — Vue 3 Composition API

```html
<script setup lang="ts">
import { ref, onMounted } from 'vue'

// ref() 创建响应式数据
// 当 notebookList.value 改变时，页面会自动重新渲染
const notebookList = ref<API.NotebookVO[]>([])

// onMounted 生命周期钩子：组件渲染到 DOM 后立即执行
// 用于页面初始化时加载数据
onMounted(() => {
  loadNotebookList()  // 页面加载后立即获取笔记本列表
})

// async/await 异步函数
const loadNotebookList = async () => {
  const res = await listNotebook()  // 调用后端接口
  if (res.data.code === 0) {
    notebookList.value = res.data.data || []  // 更新数据，页面自动刷新
  }
}
</script>
```

### 页面布局说明

```
┌──────────────────────────────────────────┐
│  我的笔记本                    [+ 新建]  │
├──────────────────────────────────────────┤
│ ┌─────────┐ ┌─────────┐ ┌─────────┐    │
│ │ + 新建  │ │ 📖 RAG │ │ 📖 AI   │    │
│ │         │ │ 学习笔记│ │ 对话记录│    │
│ │         │ │ 3个来源 │ │ 1个来源 │    │
│ └─────────┘ └─────────┘ └─────────┘    │
└──────────────────────────────────────────┘
```

- **Grid 布局**：使用 CSS Grid 实现响应式卡片排列，`minmax(280px, 1fr)` 确保卡片最小 280px，自动换行
- **Hover 效果**：鼠标悬浮时卡片上移 2px + 蓝色阴影，增加交互感
- **删除确认**：使用 Ant Design Vue 的 `a-popconfirm` 组件，防止误删

## Step 10 — 创建笔记本弹窗组件

### 做什么？

封装一个可复用的弹窗组件，用于创建笔记本。

### 新建文件

📄 **src/components/notebook/CreateNotebookDialog.vue**

### 核心代码解析 — 组件通信

```html
<!-- 父组件（HomePage）使用子组件 -->
<CreateNotebookDialog
  :visible="createDialogVisible"           <!-- 传入 prop 控制显示 -->
  @update:visible="val => visible = val"   <!-- 监听子组件关闭事件 -->
  @success="onCreateSuccess"               <!-- 监听创建成功事件 -->
/>
```

```typescript
// 子组件内部
const props = defineProps<{
  visible: boolean  // 接收父组件传入的属性
}>()

const emit = defineEmits<{
  (e: 'update:visible', value: boolean): void  // 通知父组件更新 visible
  (e: 'success'): void                         // 通知父组件创建成功
}>()
```

### Props / Emits 通信流程

```
父组件                          子组件
  │                               │
  │── :visible="true" ──────────→│  (打开弹窗)
  │                               │
  │                               │── 用户填写表单 → 调用后端API
  │                               │
  │←── emit('success') ──────────│  (创建成功)
  │                               │
  │── :visible="false" ─────────→│  (关闭弹窗)
  │── loadNotebookList() ────────│  (刷新列表)
```

## 本阶段知识点总结

| 知识点 | 说明 |
|--------|------|
| **MVC 分层架构** | Controller → Service → Mapper → Database |
| **Entity / DTO / VO** | 不同层次使用不同的数据对象，实现解耦和安全 |
| **MyBatis-Flex BaseMapper** | 无需写 SQL 即可完成基础 CRUD |
| **逻辑删除** | `@Column(isLogicDelete = true)` 自动处理 |
| **QueryWrapper** | 链式构建查询条件，避免拼接 SQL 字符串 |
| **数据隔离** | 每个接口校验用户归属权，防止越权访问 |
| **Vue 3 Composition API** | `ref()`、`onMounted()`、`defineProps()`、`defineEmits()` |
| **组件化** | 弹窗封装为独立组件，通过 Props/Emits 通信 |
| **CSS Grid 布局** | 响应式卡片排列 |
| **TypeScript 类型** | 前后端数据结构对齐，编译时发现错误 |
